Overview

Wrapper class for the underlying C++ GenericContainer. It offers convenience methods for setting/getting keys.

Concepts

The Container class is intended to provide a safe and robust way for wrapping problem data into a single container. It has a direct mapping with standard Hash and thus to the C++ GenericContainer class internally used by the OCP/DS libraries for loading problem data.

Caution: by default, key names are mapped to dynamic method names beginning with a capital letter, so that there is no risk of clashing with existing names, which in Ruby always start with lowercase letter.

Examples:

Minimal usage:

cont = Mechatronix::Container.new
cont.A           = 10
cont.Another_key = 20
p count.keys   #=> [:A, :Another_key]
p cont.to_hash #=> {:A => 10, :Another_key => 20}

Change key filtering criterion (match):

cont = Mechatronix::Container.new
cont.match = /^k/
cont.k_name    = "My data"
cont.k_val     = 20.0
cont.k_derived = cont.key_val * 2
p count.keys #=> [:k_name, :k_val, :k_derived]

Change key converter:

cont = Mechatronix::Container.new
cont.converter = :to_s
cont.Name    = "My data"
cont.Val     = 20.0
cont.Derived = cont.Val * 2
p count.keys #=> ["Name", "Val", "Derived"]

def method_missing name, *args, &block
  unless name.to_s.match(@match)
    raise RuntimeError, "Undefined method #{name}"
  end

  case name.to_s
  when /=$/
    key = name.to_s.chop.to_sym
    @content[key.send(@key_converter)] = args.first
  else
    return @content[name.send(@key_converter)]
  end
end

def self.[](hsh)
  raise ArgumentError, "Need a Hash" unless hsh.respond_to? :to_hash
  m = self.new
  hsh.to_hash.each do |k,v|
    m.send("#{k}=".to_sym, v)
  end
  return m
end
