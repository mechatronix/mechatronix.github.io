+ (Object) mechatronix(name = nil) {|data| ... }

Convenience method for setting problem data into the Mechatronix.content instance of Mechatronix::Container.

Examples:

Simple example:

mechatronix("My Model") do |data|
  data.LU_method = LU_automatic
end

which is a shortcut to:

Mechatronix.content = Mechatronix::Container.new("My Model")
Mechatronix.content.LU_method = LU_automatic

Parameters:

  • name (String) (defaults to: nil) —

    optional name of container

Yields:

  • (data) —

    a block

Yield Parameters:

  • a (Mechatronix::Container) —

    pointer to the container

def self.mechatronix(name=nil, &block)
  unless (Mechatronix.content && Mechatronix.content.kind_of?(Mechatronix::Container)) then
    Mechatronix.content = Mechatronix::Container.new(name) 
  end
  yield Mechatronix.content
end

- (String) shell(cmd, input = nil)

Runs a command in shell and returns the result as a String. This command is portable and should work on Mac, Linux, and Windows (but the must be supported by the OS!)

Examples:

List the current directory (on Max or Linux):

shell("ls #{dir}") #=> a String containing the list of files in `dir`

List the current directory (on Windows):

shell("dir #{dir}") #=> a String containing the list of files in `dir`

Parameters:

  • cmd (String) —

    the command to be executed

  • input (String) (defaults to: nil) —

    the standard input to be passed to cmd.

Returns:

  • (String) —

    if input=nil, an Array with the command output and the command exit code, otherwise a Fixnum representing the command exit code
